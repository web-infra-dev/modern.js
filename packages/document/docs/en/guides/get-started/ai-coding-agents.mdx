# Modern.js For AI

Modern.js provides tooling for AI agents (Claude Code, Codex, Cursor, etc.) to help developers complete **feature development, dependency upgrades, and version migration** more efficiently. For **developers using Modern.js**, there are two kinds of capability:

- **llms.txt** — the official docs knowledge source (large, auto-generated) for AI tools to retrieve facts.
- **Skills** — installable, on-demand multi-step workflows (with scripts/state).

A one-line rule: look facts up in **llms.txt**; use a **Skill** for multi-step verifiable workflows.

> About `AGENTS.md`: it is an internal asset of the **Modern.js repository itself**, used by AI agents that **maintain this repo**. **It is not for your application project** — `@modern-js/create` does not generate it, and you do not need one in your own project.

## llms.txt

Modern.js docs follow the [llms.txt specification](https://llmstxt.org/), auto-generated by [`@rspress/plugin-llms`](https://rspress.rs/plugin/official-plugins/llms), accessible via `/llms.txt` or `/llms-full.txt`:

- Index: [`https://modernjs.dev/llms.txt`](https://modernjs.dev/llms.txt)
- Full text: `https://modernjs.dev/llms-full.txt` (large — fetch on demand)

Most "what is this API / config" questions can be answered from llms.txt. Just let your agent retrieve it on demand; no need to copy docs into your project.

## Skills

Skills are on-demand AI capabilities following the [Agent Skills open standard](https://github.com/vercel-labs/next-skills). Planned user-facing Skills:

| Skill | Identifier | Description |
|---|---|---|
| Upgrade to v3 | `modernjs-migrate-to-v3` | Migrate a v2 app to v3: safe rewrites + manual checklist + migration report |
| Enable features | `modernjs-feature-enable` | Enable BFF / SSG / RSC / Tailwind / Module Federation on demand |
| Dependency audit | `modernjs-dependency-audit` | Phantom deps, circular deps, duplicate versions, install-size attribution |

> Skills are **not force-installed and never installed implicitly** — you install them explicitly. Some Skills are still under implementation.

## Installing Skills

User-facing Skills are installed explicitly via the standalone `@modern-js/skills` package:

```bash
# List installable Skills
npx @modern-js/skills list

# Install a Skill into your agent directory; supports --target=claude|codex|cursor|all
npx @modern-js/skills add modernjs-dependency-audit --target=claude
```

After installation, the Skill appears in the corresponding agent directory (e.g. `.claude/skills/` for Claude Code, `.agents/skills/` for Codex) and can be triggered there.
