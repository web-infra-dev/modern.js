# Modern.js For AI

Modern.js provides a toolkit for AI agents (Claude Code, Codex, Cursor, etc.) to help developers efficiently complete **feature development, dependency upgrades, and version migration**. The toolkit is made of three parts, each with its own job:

- **llms.txt** — the official docs knowledge source (large, auto-generated) for AI tools to retrieve facts.
- **AGENTS.md** — always-on project guidance (thin): project-level constraints and routing.
- **Skills** — installable, on-demand multi-step workflows (with scripts/state).

A one-line rule: look facts up in **llms.txt**; put project-level constraints in **AGENTS.md**; turn multi-step verifiable workflows into a **Skill**.

## Two audiences, two paths

| You are… | Entry point |
|---|---|
| **Maintaining the Modern.js repo** | Read the repo root `AGENTS.md` (commands, no-touch zones, version priority, Skill routing); sync/use the maintainer Skills under `skills/maintainer/*` as needed |
| **Building an app with Modern.js** | Let your agent retrieve [`modernjs.dev/llms.txt`](https://modernjs.dev/llms.txt) first; for workflows like migration / dependency audit, **explicitly install** a user-facing Skill |

> Note: `@modern-js/create` does **not** generate `AGENTS.md` / `CLAUDE.md` in your project by default, nor does it auto-install Skills. User-side AI assets always go through explicit entry points (see "Quick integration" below).

## llms.txt

Modern.js docs follow the [llms.txt specification](https://llmstxt.org/), auto-generated by [`@rspress/plugin-llms`](https://rspress.rs/plugin/official-plugins/llms), accessible via `/llms.txt` or `/llms-full.txt` for AI tool integration:

- Index: [`https://modernjs.dev/llms.txt`](https://modernjs.dev/llms.txt)
- Full text: `https://modernjs.dev/llms-full.txt` (large — fetch on demand)

Most "what is this API / config" questions can be answered from llms.txt alone.

## Skills

Skills are on-demand AI capabilities following the [Agent Skills open standard](https://github.com/vercel-labs/next-skills). The planned Modern.js Skills (coming soon):

| Audience | Skill | Identifier | Description |
|---|---|---|---|
| User | Upgrade to v3 | `modernjs-migrate-to-v3` | Migrate a v2 app to v3: safe rewrites + manual checklist for complex items + migration report |
| User | Enable features | `modernjs-feature-enable` | Enable BFF / SSG / RSC / Tailwind / Module Federation, etc. on demand |
| User / Maintainer | Dependency audit | `modernjs-dependency-audit` | Circular deps, phantom deps, duplicate versions, install size & time attribution |
| Maintainer | Issue triage | `modernjs-issue-triage` | Issue classification, minimal-repro suggestions, labels and reply drafts |
| Maintainer | PR review | `modernjs-pr-review` | changeset, public API changes, doc sync, CI conventions |

> Skills are **not force-installed and never installed implicitly**. The Skills above are under implementation; install entry points will arrive in later releases.

## Quick integration (coming soon)

User-facing Skills are installed explicitly via the standalone `@modern-js/skills` package:

```bash
# Optional: generate a lightweight AGENTS.md in your project
npx @modern-js/skills init

# Install a single Skill; supports --target=claude|codex|cursor|all
npx @modern-js/skills add modernjs-migrate-to-v3
```

The single source of truth for maintainer Skills lives in this repo at `skills/maintainer/*`; tool directories like `.claude/skills` and `.agents/skills` are generated by a sync script (derived artifacts, not hand-written sources).
