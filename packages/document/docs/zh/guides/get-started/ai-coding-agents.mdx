# 在 AI 编码助手中使用 Modern.js

Modern.js 为 AI 编码助手（Claude Code、Cursor 等）提供了一套配套资产，帮助你在 AI Agent 中**稳定、准确**地开发。本文说明这套资产的边界与使用方式。

## 三类资产的边界

Modern.js 的 AI 基建由三类资产组成，**各司其职、互相协作，而非三选一**：

| 资产 | 职责 | 何时使用 |
|---|---|---|
| **llms.txt** | 知识索引（厚、由文档自动生成） | 查 API / 配置 / 概念等事实性问题 |
| **AGENTS.md** | 项目约束 + 路由（薄、常驻上下文） | 告诉 Agent「在这个项目里要遵守什么、去哪找」 |
| **Skills** | 可验证的多步骤流程（带脚本/状态） | 迁移、依赖体检、issue 分诊等复杂工作流 |

一句话判定：

- 能用一段文档回答的事实 → 查 **llms.txt**。
- 跟「这个项目」强绑定的约束 → 写进 **AGENTS.md**。
- 多步骤 + 需脚本/判断 + 高频痛点 → 做成 **Skill**（其余沉淀进 AGENTS.md 或文档，避免滥造 Skill）。

## llms.txt：官方知识索引

Modern.js 文档站已内置 [`@rspress/plugin-llms`](https://rspress.rs/plugin/official-plugins/llms)，自动产出：

- 索引：[`https://modernjs.dev/llms.txt`](https://modernjs.dev/llms.txt)
- 全文：`https://modernjs.dev/llms-full.txt`（体积较大，建议按需取片段，不要整份塞进上下文）

在你的项目里，**只需在 `AGENTS.md` 中引用 `llms.txt` 链接**，让 Agent 按需检索，而不要把厚文档复制进仓库。

## AGENTS.md / CLAUDE.md：项目常驻指引

使用 `@modern-js/create` 创建项目时，默认会生成一份**轻量** `AGENTS.md` 与一份指向它的 `CLAUDE.md`，内容只包含：

- 项目命令（dev / build / serve / lint）
- 禁改区（lockfile、dist、node_modules）
- 版本优先级（让 Agent 的回答匹配你的 Modern.js 版本）
- 文档与 Skills 的路由

保持 `AGENTS.md` **精简**是有意为之：常驻上下文越短越可靠，详细知识交给 llms.txt，复杂流程交给 Skills。

### 创建项目时的相关参数

```bash
# 默认：生成轻量 AGENTS.md / CLAUDE.md，并提示推荐 Skills（不自动安装）
npx @modern-js/create my-app

# 跳过生成 AGENTS.md / CLAUDE.md
npx @modern-js/create my-app --no-agents-md

# 控制 Skills 模式：none | recommended | custom（默认 recommended）
npx @modern-js/create my-app --skills=none
```

- `--skills` **只做选择/提示**，不会在创建阶段隐式安装任何 Skill，也不会拉取外部包。
- 以 `--sub` 创建的子项目会**继承 monorepo 根目录**的 `AGENTS.md`，不在子包内重复生成。

## Skills：可验证的多步骤流程

Skills 遵循 [Agent Skills 开放标准](https://github.com/vercel-labs/next-skills)，是按需触发的工作流。Modern.js 的 Skills **默认不强装、不隐式安装**，规划中的首批包括：

- `modernjs-migrate-to-v3`（用户向）：v2 → v3 迁移。首版只做安全改写，复杂项进入人工清单并产出迁移报告。
- `modernjs-dependency-audit`（维护者/用户共用）：循环依赖、幽灵依赖、重复版本、安装体积与耗时归因。
- `modernjs-issue-triage`（维护者向）：issue 分类、最小复现建议、标签与回复草稿。

> 这些 Skill 正在实现中，安装入口将随后续版本提供。
